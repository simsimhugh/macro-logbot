## 3. **Critical findings** — baseline 의 실제 의미

### F1 — backend HTTP 500 cold start (N1 E001~E004)
**원인**: LM Studio Gemma 3 12B 의 **context window 초과** (`OpenAIError: 400 — Context size has been exceeded`).

```
File "/usr/local/lib/python3.14/site-packages/litellm/llms/openai/openai.py", line 990
litellm.llms.openai.common_utils.OpenAIError: Error code: 400 -
  {'error': 'Context size has been exceeded.'}
```

agent loop 가 stderr + system prompt + tool 응답 + reasoning 누적되면서 context limit 초과. **N1 의 첫 4 case fail 의 진짜 원인**. N1<N2<N3 추세는 KB 누적/학습 아니라 **단순 cold start**.

→ **task-AGENT-010** (신규): backend agent loop context 모니터링 + auto-truncate.

### F2 — agent tool 호출 실패 (workspace boundary)
30/30 case 의 backend_response 분석:
- **모든 case 가 `report.location == None`**
- 자연어 답 시작 부분이 일관: *"죄송합니다. 현재 작업 디렉토리 밖에 있는 경로에 접근할 수 없습니다."* / *"`read_file` tool 에서 `path outside working directory` 에러"* / *"`grep_codebase` 도 실패했습니다"*

**원인**: agent (Gemma 3 12B) 가 traceback 의 `/tmp/poc-E001-xxx/snake.py` 경로를 `read_file` tool 호출 시도. backend tool 의 workspace boundary check 가 거절 (`/tmp/poc-*` 가 backend `/app` workspace 밖). agent 가 fallback 으로 stderr 만 보고 자연어 추측.

→ **task-TOOL-001** (신규): backend tool `read_file` / `grep_codebase` 의 workspace 정책 — PoC 측정 시 `/tmp/poc-*` 도 허용 (또는 inject.py 가 backend workspace 안으로 copy).

### F3 — structured Report 부재 (자연어 fallback)
agent 의 의도된 동작 (spec §5.2 `analyze_event` Report schema):
```yaml
report:
  root_cause: <structured>
  location: {file, function, line}
  fix_hint: <structured>
  confidence: 0~1
```

실제 결과: 30/30 case 가 `location: None`, `root_cause`/`fix_hint` 가 자연어 사과문 그대로. crystallize 단계가 LLM 의 자연어 답을 그대로 박음.

**진짜 자율해결률 = 0%** (structured 답 0 개). 1-A 의 50%~80% 는 **stderr keyword 우연 match** (analysis 자연어 안에 ground truth keywords 가 우연 등장).

→ **task-AGENT-011** (신규): crystallize 단계가 자연어 → structured 변환 강화 (현재는 LLM 답을 그대로 박음).

## 4. variance 분석

### N1<N2<N3 추세의 진짜 원인

| 가설 | 결론 |
|---|---|
| KB 누적 학습 (`task-KB-002`) | ❌ **false**. backend `MACRO_LOGBOT_KB_AUTO_ARCHIVE` env 꺼짐 (default off). KB write 없음 |
| Session DB 누적 | ❌ false. `evaluate.py` 가 `session_id` payload 미포함 — 매 case 신규 session |
| Backend singleton state | ❌ unlikely. fresh request 마다 새 LangGraph state |
| **Backend cold start (Gemma context)** | ✅ **확정**. N1 의 E001~E004 HTTP 500 (Context size exceeded). N2/N3 부터 안정 |
| **Sampling stochasticity (temperature 미지정)** | ✅ **유력**. `evaluate.py` 가 `temperature/seed` payload 미포함 → LiteLLM default → 매 회 답 다름 |

### case 별 catastrophic variance

| case | pattern | 추정 |
|---|---|---|
| E004 | 0.0 → 0.58 → **1.0** | N1 cold start fail, N3 우연 file+line+keyword 모두 hit |
| E008 | 0.475 → 0.4 → 0.075 | N3 regression — infinite loop case, agent 의 추측이 stderr 의 다른 keyword 잡음 |
| E009 | 0.925 → **0.0** → 0.85 | N2 한정 catastrophic — temperature stochasticity 추정 |

→ **N=3 은 variance 확정에 너무 작음**. N=10+ 필요. 또는 deterministic (`temperature=0, seed=42`) 강제 후 N=3 충분.

## 6. KB design flaw (사용자 critical 지적, 2026-05-20 02:30)

### 발견
`src/macro_logbot/app.py:311-324` `_kb_auto_archive`:
```python
case = ArchivedCase(
    case_id=str(uuid4()),
    error_signature=report.root_cause[:80],
    category="auto/poc",
    root_cause=report.root_cause,        # 분석가 답 그대로
    location=location,
    fix_hint=report.fix_hint,
    confidence=report.confidence,        # 자체 confidence (self-rated)
    source="poc",                        # spec 의 verified-master 분리 없음
)
kb_store.add(case)
```

**문제**: verified 검증 없이 분석 결과를 KB add. self-bootstrapping risk — 잘못된 분석이 후속 case 답 망침.

### spec 정합
`docs/design/02-설계문서.md:231,252,253,261` 에 **이미 명시**:
- `source: poc | production | verified-master`
- 검색 시 `verified-master` 가중치 우선
- PoC 매트릭스 정답 일치 시 자동 `verified-master` 승격

→ spec 은 정책 정의됨. **구현이 미완 (verifier 승격 hook 없음)**. `task-KB-002` line 259 가 이미 잔여 명시: "verifier 승격 (`verified-master`) hook → 별도".

→ **task-KB-004** (신규): `ArchivedCase` tier 정책 구현 (draft / verified / verified-master).
